Introduction

If you'd like to check The Number Game out for yourself before reading this blog post (which I highly recommend doing), go ahead and check out the live demo below! You can also take a look at the repo for all of my files related to this project, including both the new and the old JS files.

  • Live Demo
  • Code Repo

The Number Game is a number guessing game in which the player must guess the randomly-generated winning number in the least amount of guesses possible. After each guess, the game lets the player know whether they were too high, too low, or just right.

I decided to do this project when going down the list of projects on this webpage. The original instructions were as follows:

Write a guessing game where the user has to guess a secret number. After every guess the program tells the user whether their number was too large or too small. At the end the number of tries needed should be printed. It counts only as one try if they input the same number multiple times consecutively.

This was first completed some number of months ago. Within the last week, however, I decided to revisit this project (in the hopes of writing the Project Breakdown, actually), and realized something: The code was unnecessarily way too long! I had put class changes on their own individual lines rather than grouping added classes and removed classes together, resulting in painful-looking code like this:

After the onlyUnique function are these two event listeners.

Next we begin this if condition, the first part of which you're seeing above. This section is essentially saying "For all player guesses that are below the winning number and above the number 0" (or, in other words, for every low guess):

  • guesses.push(userGuess) - The player's guess number is pushed into the guesses array.
  • result.classList.remove('high'), result.classList.remove('error') - The classes 'high' and 'error' are removed from the results text.
  • result.textContent = "..." - Updates the text content of the results text to inform the player that their guess was too low.
  • result.classList.add('low') - Adds the correct class, 'low', to the same reults text.
  • newNum
    • newNum = document.createElement('span') - A new <span> element is created
    • newNum.classList.add('low') - This span element is given the 'low' class to ensure it's the correct color
    • newNum.innerText = guess.value + ' ' - And finally, the span's inner text is set to the user's current guess value, followed by a space.
  • user2.appendChild(newNum) and user2.appendChild(slash) - Append the guessed number with a space, and then a slash, to its correct place.

After that, we make sure that we clear any old classes from the background and the buttons, and we add the appropriate class to these elements.

  2. Above the number 100
  3. Blank

then the game's elements turn red, and the player gets a little error message telling them to enter an appropriate guess.

Let's go through each of these briefly.

  • The unique array we discussed earlier is back to finish its job. It Filters out only the unqiue guesses from the guesses array.
  • The counting element is given text content of the total number of the user's unique guesses, which is done by taking the unique array's length (which is why we created a separate array).
  • We also append the user2 element to the page, and finish up the function by resetting the user's guess number to nothing as well as the value of the input box to nothing.

Part 5 - refreshPage

Our final part of this code is the refreshPage() function, which is executed when the player activates the Reset button. 

function refreshPage() {
    if (guess.disabled) {
        location.reload();
    } else {
        if (confirm("Are you sure that you want to reset the game?")) {
            location.reload();
        }
    }
}

This function is a little more complicated than merely refreshing the page, however. Instead, it contains an if condition that tests whether the input box is disabled (recall that it is disabled once the game is over). If this is the case, the page refreshes automatically. However, if the input box isn't disabled, this prompts a confirmation box to appear, asking the player whether they're sure they want to restart the game.

This was included after I added the keyboard shortcuts. I didn't want there to be incidents where the player accidentally presses the shortcut (which, as a reminder, is simply Shift + R), restarting their game immediately. I made this an if condition because there's no reason to have the confirmation box appear when the game is over anyway.

And that concludes the old code! How painful. Let's move on, shall we?

The New Code (jQuery)

Just as a quick recap, we are discussing the JavaScript-based code for my browser-based number guessing game, The Number Game. In the previous section, we went through the original code, which is entirely too long. I've made the decision to rewrite the entire thing in jQuery because a) I know it will reduce the length of it tremendously, and b) I need to practice jQuery to learn it anyway.

I do already know a few things about jQuery, such as how to use the "selectors" at the beginning of each line of code, and that you can chain several methods together for one selector, which I think is pretty cool. I'm hoping that we can reduce the 152 lines of code down to under 100, which I don't doubt for a moment.

I've already gone ahead and written the jQuery code. To clarify, although I do believe that I had a natural "talent", it didn't come without errors. My first attempt failed due to some misplaced syntax errors, and I had to google a few things, which I'll discuss as we go through it.

For comparison's sake, I'm going to place pieces of the old code above the new code. Let's see what I came up with, shall we?
