Code Discussion

I'm about to do something that may turn you off, dear reader, but you're going to have to kind of get over it because it's something I that rarely do, which is to brag.

Okay? Okay, cool. So here it goes:

I am so fucking smart!

/ end brag

In all seriousness, I am extremely proud of myself. I was working on my main portfolio page today and remembered that I wanted to implement a "Categories" system. To quickly summarize, there would be a series of buttons at the top of the page, and when a button is clicked, it would filter the portfolio projects to only show the ones belonging to the button's category. My categories would be based off of coding languages, frameworks, etc., so the buttons would include HTML, CSS, JavaScript, jQuery, Bootstrap, etc.

Going into this, I wasn't quite sure how to implement such a system. I knew it couldn't be complicated, but I've never done it before.

Relevant side story time!

I used datatypes yesterday to make my portfolio system work. Hovering over one of the project images pulls up the project's text, a small description, and two buttons - one with a magnifying glass icon, and one with an eye icon. When you click on the button with the magnifying glass icon, the project image and its caption appears in the modal that pops up.

I actually had a bit of difficulty with this task. No matter what I did, the only modal picture popping up was the one for the very first portfolio project. I kid you not, I think I spent hours on this, but that was mainly because I was trying to use JavaScript to automate the whole thing rather than to attach individual IDs to each and every element of each and every project div/modal.

I was actually (mostly) successful in this task shortly after beginning. I was able to get each individual button to get their respective modals to appear, and each modal had its own X button in the corner to close it. This was neatly packaged into a single for loop. If I could get this to work in one loop, I was certain I could also get the image and its caption to work in that same loop.

The thing that caused me hours of anguish was getting the individual images to appear in their modals. For the life of me, I couldn't accomplish this. I did a lot of research on Google, but a lot of the solutions were for clicking on the image and having it appear in a modal, which works easily enough with the use of this. What I was trying to do was to click on a button and have a unique image appear for each modal.

My thought process was that, since I was using a loop, I needed to have a portfolio image array and a modal image array. I would then use JavaScript to pull all of the image src's from the page (rather than type them in individually) via the portfolio image array, and to then put them into the modal image array. I thought that if I could get that far, I could figure out how to display the image in its modal easy peasy.

I tried several (way too many) variations of trying to get something like this to work:

modalImages[i].src = portfolioImg[i].src;
modalImages[i].src = portfolioImg[i].getAttribute('src');
modalImages[i].setAttribute('src', portfolioImg[i].getAttribute('src'));

I also tried separate functions, separate for loops, other methods, etc. I spent more time doing this than if I were to just do the deed manually, but this was for the sake of good coding, and my portfolio's future. After all, I didn't want to have to manually do this every time I changed up my projects.

After waaaay too much time had been spent on this conundrum (I hadn't even gotten to the Captions part of it), I finally figured out that to do this, I could use data keys, which I've never actually used before. Bonus: I could knock out two birds with one stone and use data keys for both the image src and the caption. So I created two datatypes - data-alt for the caption and data-src for the image, placing them as attributes of the portfolio image itself to keep things simple for myself when coding new projects into the portfolio. To my relief, it finally worked! My final code is as such:

let modal = document.querySelectorAll('.modal');
let btn = document.querySelectorAll('.modalbtn');
let span = document.getElementsByClassName('close');
let captions = document.getElementsByClassName('caption');
let portfolioImg = document.getElementsByClassName('portfolio-img');
let modalImages = document.getElementsByClassName('modalimg');

if (let i = 0; i < modal.length; i++) {
    btn[i].onclick = function() {
        modal[i].style.display = "block";
        captions[i].textContent = portfolioImg[i].dataset.alt;
        modalImages[i].src = portfolioImg[i].dataset.src;
    }
    
    span[i].onclick = function() {
        modal[i].style.display = "none";
    }
}

Why did I just tell you that rather long story, and in the middle of the main point of this blog post at that? Well, as I said, it's relevant. Cut to today, and I have datatypes on the mind. To me, although I'd known about them (albeit vaguely) in the past, remember that I'd never used them before. It's a little bit like a whole new world of possibility has opened up! So, with said datatypes on the mind, I thought maybe I could use them to make this Categories system work.

I created a few of the category buttons - specifically HTML, CSS, and JavaScript - and attached a unique ID to each. Then I went into script.js and typed:

const categoryHTML =  document.getElementById('category-HTML');
var projects = document.getElementsByClassName('content');

categoryHTML.addEventListener('click', function() {
    projects.forEach((project) => {
        if (project.dataset.tag.includes())
    })
})

Then I paused. How the heck did one call upon just one part of a datatype? I was considering the use of something like .includes(), but that's a Ruby thing, and the datatype isn't a string or an array. I erased the .includes( ) bit and pondered. And when pondering didn't get me any further, I headed for Google.

I typed in queries like "javascript dataset includes", "javascript check if dataset includes value", "javascript custom attribute", and "javascript html css multiple values in one dataset". Unsurprisingly, I didn't find an answer.

The problem with this initial idea of mine is that, as far as I know, you can't simply list a bunch of things under one datatype and then call upon them individually via some method or function. For example, if I had data-category="HTML, CSS, JS, JQ, BS", I couldn't use JavaScript to only call on the ones that include HTML within that datakey. Using multiple datakeys (one for each category) was out of the question because a) it's tedious, b) there will be a lot more categories in the future, and c) it defeats the purpose of trying to script the whole thing.

Don't worry, this story does have a happy ending: 4 minutes into this task, my lightbulb finally went off! Hello, Sara, dumbass, you ever hear of a thing called CSS classes?! Classes are able to do exactly what I was trying to force datatypes to do. You can give an element multiple classes and call upon said classes individually via JavaScript. (I'm just relieved that I didn't spend more than 5 - 10 minutes before I realized that datatypes weren't the way to go.)

With a better idea in mind, I set off to try it out, and this is what I came up with:

const categoryHTML = document.getElementById('category-HTML');
const categoryCSS = document.getElementById('category-CSS');
const categoryJS = document.getElementById('category-JS');
const categoryJQ = document.getElementById('category-JQ');
const categoryBS = document.getElementById('category-BS');
const showAll = document.getElementById('show-categories');
const portfolioDivs = document.querySelectorAll('.content');
const categoryBtns = document.querySelectorAll('.portfolio-category');

categoryHTML.addEventListener('click', function() {
    categoryBtns.forEach((categoryBtn) => {
        if(categoryBtn.classList.contains("category-active")) {
            categoryBtn.classList.remove('category-active');
        }
    });
    
    portfolioDivs.forEach((portfolioDiv) => {
        if(portfolioDiv.classList.contains("HTML")) {
            portfolioDiv.classList.remove("portfoliohide");
            portfolioDiv.classList.add("portfolioshow");
        } else {
            portfolioDiv.classList.remove("portfolioshow");
            portfolioDiv.classList.add("portfoliohide");
        }
    });
    
    categoryHTML.classList.add("category-active");
});

showAll.addEventListener('click', function() {
    categoryBtns.forEach((categoryBtn) => {
        categoryBtn.classList.remove("category-active");
    });
    
    portfolioDivs.forEach((portfolioDiv) => {
        portfolioDiv.classList.remove("portfoliohide");
        portfolioDiv.classList.add("portfolioshow");
    });
    
    showAll.classList.add("category-active");
});

Although this was a rather long and confusing story, I wanted to express my happiness at having figured out a whole little system so quickly. I'm sure to some of you more seasoned programmers reading this, it probably doesn't seem like a big deal. The main reason I'm excited about it is because I feel like it's a sign that I really have the "frontend" thing down. I mean, I've had HTML and CSS down for months now, but when I first began JavaScript, I really struggled with it for a while because I failed to see how and where it could be applied (that's one of the biggest problems I have with most JavaScript courses, but that's something for another blog post entirely). Now that I know and now that I can implement it (frontend-wise, anyway) pretty easily, I'm probably going to concentrate on something else. Yay!
